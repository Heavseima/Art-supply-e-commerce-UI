## Palette & Theming
- Canvas Cream `#FAF9F6` (--canvas-cream), Ink Charcoal `#1C1A1A` (--ink-charcoal), muted `#6b6764`, hairline `rgba(28,26,26,0.12)`, accent terracotta `#B4533A`
- Tailwind v4 custom properties exposed via `@theme inline` in `src/app/globals.css`
- Three recurring off-palette hardcoded bg values used inline across components: `#f1efe9` (image placeholder), `#f4f2ec` (category strip bg), `#efece5` (skeleton pulse bg) — these should be CSS vars but are not yet

## Primitive Locations
- UI atoms: `src/components/ui/` — Button, Price, StockBadge, SectionHeading, QuantityStepper, ProductCard, ProductGrid, GridSkeleton, Hero, Logo, TextField
- Modules: `src/components/modules/` — ElegantNavbar, CartDrawer, CategoryFilter, FeaturedCollection, RelatedProducts, PurchasePanel, AddToCartButton, CartView, CheckoutForm, SiteFooter
- Hook: `src/hooks/useCart.ts` (`"use client"`)
- Types: `src/types/api.ts` (Product, Category, CartLine, ResolvedCartLine, ShippingDetails, ProductId, CategoryId)
- Utils: `src/utils/catalog.ts` (seed data), `src/utils/products.ts` (cached fetch engines), `src/utils/format.ts` (formatPrice, stockLabel), `src/utils/cartStorage.ts` (readCart, writeCart, CART_EVENT, CART_STORAGE_KEY)

## Next.js 16.2.7 Caching Model (CONFIRMED from local docs)
- `cacheComponents: true` is REQUIRED in `next.config.ts` to enable `"use cache"` directive — confirmed enabled in this project
- `"use cache"` used in `src/utils/products.ts` on all 5 data functions, paired with `cacheLife(profile)` and `cacheTag(...)` — correct usage
- `PageProps<'/route'>` is a globally available helper (no import needed), generated by `next typegen` / `next build`, stored in `.next/types/routes.d.ts` — correctly used in products/page.tsx and products/[slug]/page.tsx
- `searchParams` and `params` are Promises in Next.js 16 — must be awaited

## Approved Dependencies
- `next@16.2.7`, `react@19.2.4`, `react-dom@19.2.4`
- `lucide-react@1.17.0` — added to package.json (was not in original), used only in AddToCartButton.tsx and PurchasePanel.tsx for the `Check` icon confirmation state

## Known Issues Found in Audit (2026-06-06)
- `CategoryFilter` marked `"use client"` unnecessarily — only uses `Link`, no hooks or interactivity; should be server component
- `AddToCartButton` is declared but never imported or used in any page/component; dead code
- `window.setTimeout` in PurchasePanel and AddToCartButton lacks cleanup — timer ID not stored/cleared, causes state update on unmounted component if user navigates away within 1.6s
- `CartDrawer` missing focus trap and `inert` attribute on background content when open — keyboard users can tab through background
- Breadcrumb `<nav>` in product detail page lacks `aria-label` to distinguish it from the main navigation
- Inline hardcoded hex colors (`#f1efe9`, `#f4f2ec`, `#efece5`) should be CSS custom properties
- `resolve(catalog)` called inline during render in 3 components (CartDrawer, CartView, CheckoutForm) — not memoized at call site, though `resolve` itself is `useCallback`-wrapped in the hook (catalog reference stable from server = acceptably low risk but still a concern)
- Footer "The House" nav links rendered as `<span>` not `<a>` — not keyboard navigable, semantically wrong
- Footer newsletter `<form>` has no `name` or `id` on the submit button, no `onSubmit` handler — effectively broken
- No `loading.tsx` segments under `/products`, `/cart`, or `/checkout` — only Suspense boundaries within pages
